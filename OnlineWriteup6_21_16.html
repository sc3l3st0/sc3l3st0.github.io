OKAY, IS THIS THE PART WHERE YOU TELL ME WHAT THE HELL IS GOING ON?

The idea of having a webmaster is kinda dumb if you think about it. What does a webmaster do? He handles the online stuff when it breaks, and tries to set up everything in a way that it works smoothly. But if the stuff is already set up in a way that works smoothly, and breaks very rarely, there’s no need for a webmaster, except every few years when your shit stops working smoothly and everyone else is using brand-spanking new technology. My goal here is to try and set VoiceMale up so that we don’t need a webmaster for the next six years.

You might have noticed there were a bunch of websites in the here and now bit. I believe that if possible we should have as few websites to manage as possible. This will:

  • Save VoiceMale members time and effort

  • Be easier for us to manage
  • Be designed in such a way that we get more information online in less time

WAIT, WAIT, HOW DO YOU GET THIS THING ONLINE?
That’s another good question.

Web Hosting is pretty cool, but it can be pretty complicated. You need the following things:

  • A website name (in our case, brandeisvoicemale.com)
  • A server that runs all the time
  • “Back end” website files that secretly work in the background but don’t show up to a web browser
  • “Front end” website files that do show up to a web browser
  • Something called a DNS server, which tells a web browser that the website name should direct to the server
  • Something called an SSL certificate, which verifies the website’s identity if you want to make online payments anything else that requires security

For pretty much all of these things, there are free, cheap, and expensive ways to do them. In general, the more you pay, the better you can get your shit to turn out. So, if you want all the things I listed above, we gon need

DAT CASH YO

So right. How much of dat cash yo am I talking about? Well, that depends on the following:

  • We already have a website name and a DNS server, and they’re perfectly legitimate, so that’s all good. (well, there is a fee for the name every few years, but it’s not much)
  • Most “front end” software is free and will work the same no matter what you’re paying, so that’s all good.
  • If we want to add payment to the website, there are free options for SSL certificates which will work just fine for something as small as us, so that’s all good.
  • “Back end” software depends on the server.
  • The server depends on the server.

So, I totally lied. It only depends on the server.

So how much of dat cash yo are we talking about for the server? Well, there are several options:

  • “Static” hosting: Totally free! But the whole website is open to the public, and you can’t have any of the private features I talked about, and there’s a permanent need for a webmaster who knows how to program.
  • Unreliable “Shared hosting:” Can be but isn’t always very cheap, but is always extremely unreliable, and your site can be shut down. This would limit how many users we could have without going down, as well as just about everything else.
  • Reliable “Shared hosting:” Starts at about 12 bucks a year. This is my third favorite option. They probably won’t shut down your site. Still quite unreliable though. This would limit archive capabilities, as well as other private features I talked about.
  • “Virtual Private Server” hosting: Starts at about 24 bucks a year. This is my second favorite option. Quite reliable, but limited in terms of file space and RAM.
  • “On-Demand” hosting: Starts at about 24 bucks a year, but will most likely be more. This is my favorite option. Extremely reliable, and unlimited system specifications. As you use more stuff, you pay more.
  • “Make Your Own Website!” systems: Hundreds of dollars a month, unless a friendly alumni is paying. This is what we use right now, as well as what everyone uses. Can be reliable, but are always canned.
  • “Dedicated” hosting: Ranges from hundreds to thousands of dollars a year. This means you literally own a server and are paying somebody to store it and manage it and keep it connected.
  • “Super Legit” hosting: A whole bunch, then not much. This is where you buy the server and connect it to the internet yourself. Trust me, it’s totally called “Super Legit” hosting.
  • “Hire a Professional” systems: Don’t want to worry about any of this? Hire a professional! We’re only talking hundreds of dollars an hour. And don’t worry, it always turns out great.

I’ve heard two points so far regarding money:

  1. Do what you think is best
  2. Do it as cheap as possible

Basically, my preferred scenario is a budget of about 35 bucks per year for all online activities, which would be mostly put into server costs, and then I’d fork the budget personally for domain renewal as well as any surplus usage fees.

That being said, for anything less than that budget, I can produce a working product. It will, however, greatly affect the end result.
